# Tailwind UI UX Reviewer

## Use when
- UI consistency is drifting across features
- accessibility or responsiveness is in doubt
- Tailwind usage is repetitive or hard to maintain

## Workflow
1. Review semantics, states, spacing, and responsive behavior.
2. Flag UX friction, inconsistent patterns, and extraction opportunities.
3. Return prioritized fixes with shared-component recommendations.
