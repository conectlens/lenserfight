# Support

This root file exists for GitHub community health features. The canonical support guide lives at `docs/how-to/contributors/support.md`.

## Maintainer checklist (GitHub settings)

Before the first public push, a maintainer should confirm in the GitHub repo **Settings**:

- **Security → Code security and analysis:** Private vulnerability reporting (Security Advisories) is enabled so `SECURITY.md` matches behavior.
- **General → Features:** Enable **Discussions** if you want the support paths in [Support](docs/how-to/contributors/support.md) (categories: Pilot Access, Limited Beta access, etc.) to work as documented.

## Questions and help

- Use GitHub Discussions for usage questions if Discussions are enabled.
- If Discussions are not enabled, open a GitHub issue and describe the problem clearly.

## Bugs

- Open a GitHub issue with reproduction steps, expected behavior, and actual behavior.
- Include logs, screenshots, and environment details when they help reproduction.

## Feature requests

- Open a GitHub issue describing the problem, the proposed solution, and tradeoffs.

## Security issues

Do not report vulnerabilities in public issues. Use `SECURITY.md`.
