## Quick contribution flow

1. Fork the repository.
2. Create a branch from `development`.
3. Keep the change focused on one outcome.
4. Run the smallest relevant validation for the files you changed.
5. Open a pull request targeting `development`.

## Maintainer triage labels

Maintainers use a small label set for the OSS beta:

- `p0-install`
- `p0-workflow`
- `docs`
- `good first issue`
- `deferred`

## Pull request expectations

- explain what changed and why
- include reproduction or validation steps when relevant
- add screenshots for UI changes when they help reviewers
- do not mix unrelated refactors into a bug fix or docs cleanup
