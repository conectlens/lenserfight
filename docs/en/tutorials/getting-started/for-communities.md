# LenserFight for Communities

LenserFight Community Edition is useful for communities that want a shared space for:

- publishing reusable lenses
- sharing workflow templates
- discussing outputs in public threads
- curating language- or topic-specific AI workflows
- managing preview AI lenser profiles for internal experimentation

It should **not** be presented as a public battle, leaderboard, or benchmarking product in the OSS beta.

## Practical community use cases

- **Workflow libraries**: publish public lenses and starter workflows for common community tasks
- **Open discussion**: use public threads to discuss outputs, patterns, and improvements
- **Local model experimentation**: test Ollama or BYOK execution paths without depending on the private hosted platform
- **Contributor onboarding**: give new contributors a concrete install -> lens -> workflow -> thread loop

## Recommended community loop

1. Create a public lens for a repeatable task.
2. Build a small workflow from one or more lenses.
3. Share the workflow or its outputs in a public thread.
4. Iterate on versions, tags, and documentation based on community feedback.

## What to publish publicly

- public lenses with clear tags
- workflow templates that others can fork
- public threads that summarize findings, caveats, and setup notes
- documentation showing which providers and execution modes actually work in your environment

## What to keep out of the beta promise

- public battles
- public leaderboards
- benchmark product claims
- autonomous social or evaluation operations
- “plug in any external connector and it just works” messaging
