# Schema: xp

The `xp` schema implements LenserFight's experience point and progression system. XP is awarded for platform actions (creating content, participating in battles, voting, contributing code) and drives leveling, streaks, seasons, and badges.

XP is intentionally separate from the token economy. It cannot be purchased, traded, or converted to money. See [Token Economy](/explanation/battle-system/token-economy) for details.

## Difficulty System

Every XP rule has a `difficulty` rating. The difficulty multiplier scales the `base_xp` to produce the actual awarded `xp`:

| Difficulty | Multiplier | Use Case |
|-----------|-----------|----------|
| `easy` | 0.80x | Forum browsing, reactions, simple actions |
| `standard` | 1.00x | Prompt creation, voting, issue filing |
| `hard` | 1.50x | Battle participation, CLI usage, code reviews |
| `legendary` | 3.00x | Battle wins, main project PR merges |

**Formula**: `effective_xp = CEIL(base_xp * multiplier)`

| `difficulty` | `difficulty_enum` | Default difficulty for this app |

|--------|------|-------|
| `id` | uuid (PK) | |
| `lenser_id` | uuid | FK → `lensers.profiles` |

| `rule_id` | uuid | FK → rules |

## Core Functions

### `xp.apply()`

Central function for awarding XP. Called by triggers and RPC functions.

```sql
xp.apply(
    p_lenser_id       uuid,
    p_rule_key        text,
    p_source          source_enum,
    p_source_ref_type text,
    p_source_ref_id   uuid,
    p_app_id          uuid DEFAULT '00000000-0000-0000-0000-000000000000'
) RETURNS void
```

**Behavior:**
1. Loads the matching active rule by `action_key` (prefers exact app match over global)
2. Computes `effective_xp = CEIL(base_xp * difficulty_multiplier)`
3. Checks cooldown (time since last event)
4. Checks daily event count limit
5. Checks daily XP cap (using effective_xp)
6. Checks seasonal XP cap
7. Inserts `xp.events` row (`xp` = effective, `base_xp` = raw)
8. Upserts `xp.totals` (increment total_xp, recompute current_level)
9. Updates `xp.season_totals` for active season
10. Updates `xp.streaks` if rule has a `streak_type`

### `xp.grant_contribution_xp()`

Awards XP for open-source contributions.

```sql
xp.grant_contribution_xp(
    p_lenser_id         uuid,
    p_context           contribution_context_enum,
    p_contribution_type text,
    p_external_ref      text DEFAULT NULL,
    p_title             text DEFAULT NULL,
    p_metadata          jsonb DEFAULT NULL
) RETURNS uuid  -- contribution ID
```

Maps `(context, contribution_type)` to the appropriate rule key and calls `xp.apply()`.

### `xp.auto_create_next_season()`

Creates the next season for an app when the current one ends.

```sql
xp.auto_create_next_season(
    p_app_id        uuid,
    p_duration_days int DEFAULT 90
) RETURNS uuid  -- new season ID (NULL if current season still active)
```

### `xp.check_all_seasons()`

Iterates all active apps and calls `auto_create_next_season()`. Scheduled as a daily pg_cron job.

| Function | Purpose |

| `xp.compute_level_from(app_id, total_xp)` | Returns level number from XP total |
