# Security Policy

Do not report security vulnerabilities in public GitHub Issues.

This root file exists for GitHub’s community health features. The canonical security documentation lives in `docs/community/security.md`.

## How to report privately

Preferred: GitHub Security Advisories.

1. Open the repository on GitHub.
2. Go to **Security** → **Advisories**.
3. Choose **Report a vulnerability**.

If you cannot access the Security tab, do not disclose publicly. Ask maintainers for a safe private reporting path.

## Scope and expectations

For scope, supported versions, response timelines, and disclosure expectations, see `docs/community/security.md`.
